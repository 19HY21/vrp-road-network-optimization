# 問題定義

このドキュメントでは、本プロジェクトで扱う VRP PoC の問題設定を定義する。

## 1. 目的

実道路ネットワークを用いた配送ルート最適化が、社内 PoC として成立するかを検証する。

## 2. 背景

配送ルート最適化を机上距離や直線距離ではなく、実道路ネットワークに基づいて扱いたいという要件がある。

あわせて、OSM ベースの道路データを用いた最適化基盤が、実務に近い形でどこまで利用できるかを確認したい。

## 3. 解きたい課題

- 実道路ネットワークに基づく距離計算が安定して行えるか
- 単一都道府県内の配送を対象として VRP を成立させられるか
- 所要時間をどのような前提で扱うべきか
- OSM ベースの道路データが PoC の入力基盤として使えるか

## 4. PoC で検証したいこと

- 神奈川県全体の道路ネットワークを取得し、再利用可能な形で保持できるか
- 道路ネットワーク上で OD 間の道のり距離を安定して算出できるか
- 分割運用が必要な場合に、正本グラフからの切り出し方式で整合性を保てるか
- Google Maps 比較を通して、距離と所要時間の利用可能範囲を整理できるか
- VRP PoC の目的関数と制約条件を定義できるか

## 5. 想定利用者

- 社内で配送計画や業務改善を検討する担当者
- 最適化ロジックやデータ基盤を検証する技術者
- 将来の本実装判断や業務適用可否を確認する意思決定者

## 6. 対象範囲

- 地理範囲は神奈川県とする
- 正本ネットワークは神奈川県全体の一括取得版とする
- 配送対象は単一都道府県内配送とする
- 人数 = 台数とし、1 人が 1 台を担当する前提とする
- 目的関数は距離中心の費用最小化とする

## 7. 対象外

- 複数都道府県にまたがる配送最適化
- 都道府県間の大距離移送設計
- リアルタイム交通を直接反映した本番レベルの ETA 推定
- あらゆる業務ルールや車種別例外の完全再現

## 8. 成功条件

- 正本となる道路ネットワークを安定取得できること
- 正本グラフを再利用できること
- 県全体 routing が破綻しないこと
- OD ベースで距離を安定計算できること
- PoC の目的関数と制約条件を説明可能な形で整理できること
- 後続の VRP 実装に接続できること
